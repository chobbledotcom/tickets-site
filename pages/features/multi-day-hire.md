      # Multi-day hire

      Take bookings that span several days, such as equipment hire or venue rental. You set the duration, price each day count, and let buyers choose how long they need.
  - type: markdown
    content: |
      ## Bookings that span days

      A [daily listing](/features/daily-events/) can cover more than one
      day. You set a duration in days, and each booking occupies that
      many consecutive days from the chosen start date.

      Capacity is tracked per day, so a three-day hire starting on Monday
      uses one unit of stock on Monday, Tuesday, and Wednesday. Two
      bookings cannot overlap on the same unit.
  - type: markdown
    content: |
      ## Buyer-chosen durations and per-day pricing

      A listing can let the buyer choose how many days to book, up to the
      maximum duration you set. You price each day count separately, so
      one day might cost £50 and three days £120, and only the day counts
      you have priced are offered.

      The booking form shows the available durations with their prices,
      and the total updates as the buyer picks.
  - type: markdown
    content: |
      ## Booking windows

      Each daily listing has a booking window: a minimum number of days'
      notice a booking needs, and a maximum number of days ahead a
      booking can be made. For example, hire bookings could require one
      day's notice and be limited to 90 days ahead.
  - type: markdown
    content: |
      ## Pay what you want

      A listing can let buyers pay more than the listed price. The listed
      price acts as the minimum, and you can set a maximum, which is
      useful for donations or optional tips on top of a hire.

      The amount is validated on the server when the payment completes,
      so a buyer cannot pay less than the minimum or more than the cap.
  - type: markdown
    content: |
      ## Deliveries and collections

      Multi-day hire works with the [logistics system](/features/logistics/).
      The drop-off appears on the deliveries run sheet on the first
      booked day, and the collection appears on the last booked day of
      the hire span.

      Agents see both legs with the address, phone number, and a map
      link, and mark each one done as it happens.
